Input Basics

Button

Just a simple button, straight forward to understand.

Checkbox

Zero or more options may be selected.

Color

The way the color picker looks depends on the browser.

Date

The way the date looks depends on the browser.
Also when specifying a min/max date make sure to use the following format, YYYY/MM/DD

Datetime-Local

The way the datetime-local looks depends on the browser.
Just like the date type but it also contains a time, and the time doesnt have a specified time zone.

Email

The way the email looks depends on the browser.
Some Mobile devices recognize the email type and add the .com to the keyboard.

File

Literally behaves like all file inputs which you have used throughout your life.

Hidden

This input, as you can guess, is hidden from the user and cant be changed.
Although it can be seen if users use Dev Tools or View Source, so its best not to use as a form of security.
This is used when you want to store some data which will be included with the rest of the form without worrying about it being altered, an example can be a form id.

Image

Same functionality as the "submit" type but the difference is that it is used to replace the submit button with an image.

Month

The way the month looks depends on the browser.

Number

Allows for the input of a number.
You can set restrictions.

Password

The characters in a password field are masked as asterisks or circles.

Radio

Only one of the many options can be selected.
Remember to give all related options the same name else the user can select multiple.

Range

By default the range is 0 to 100.

Reset

Easy to guess what this does.
Has to be in a form though for it to work and it will only work for the form elements in the same form.

Search

Behaves like the "text" type but its used for search fields. I dont know, search it up. Pun unintended.

Submit

Submits the form to the form handler which is covered in the Form Lesson.

Tel

For telephone number inputs.

Text

Single line text input.
This is the default input type if non are specified.

Time

The way the time looks depends on the browser.
Time zone isnt specified.

URL

URL field may be automatically validated when submitted, depends on the browser.
On Mobile devices the keyboard adds .com when it recognizes the url type.

Week

The way week looks depends on the browser.

Input Attributes

Value

Sets an initial value for the input.

Readonly

Makes an input readable only. So it cant be changed at all.
Although if its text then it can be highlighted and copied.
And dont worry, the value in this will still be sent when submitting the form.

Disabled

Makes an input field unusable and un-clickable.
The value in this will not be sent when the form is submitted.

Size

Specifies the visible width. By default this width is 20.
This attribute also only works on the following input types: text, search, tel, url, email, and password.

Maxlength

Sets a max length of characted which can be typed.
When it reaches the max length it just stops accepting more characters so if you want to give the user an alert that they reached max length then you must use JS.
Also, as you can probably guess, only works on text accepting input types.

Min/Max

Sets the min and max values of an input.
This can only be used for the following input types: number, range, date, datetime-local, month, time, and week.
It is advisable to use both the min and max in order to create a range of legal values.

Multiple

Allows the user to insert multiple values.
This is only works for the following input types: email and file.
When using this on the file, multiple files can be selected at a time.
When using this on the email, each email is seperated by a comma(,).

Pattern

Only applicable for the following input types: text, date, search, url, tel, email, and password.
Learn more about possible patterns by learning about JS Regular Expressions.

Placeholder

Describes the expected value through a small description or an example.
This is useful when you have added a pattern.
Only works for the following input types: text, search, url, tel, email, password.

Required

Allows you to make an input necessary to fill out in order to submit the form.
Only applicable for the following input types: text, search, url, tel, email, password, date pickers, number, checkbox, radio, and file.



Step

Used to specify a "legal" number of intervals for the value.
This is only applicable for the following input types: number, range, date, datetime-local, month, time, and week.
Restrictions arent foolproof.

Autofocus

Makes the input be selected as soon as the site loads.

Height/Width

Used to specify thw width and height of an "image" input type.
Make sure to use this since not using it may bring layout issues.

List

Used when refering to a datalist element.

Autocomplete

Tells the input whether it has autocomplete on or off.
Autocomplete allows the browser to predict what the user will type and suggests options.
Only applicable to the following input types: text, search, url, tel, email, password, datepickers, range, and color.

Input FormAttributes

Form

Form allows you to specify which form an input is attached to.


FormAction

Overrides the action attribute in the form and takes the data to the overriding file for processing.
This is only applicable for the following input types: submit and image.

FormEnctype

Overrides the enctype attrubite in the form.
Only applicable for the following types: submit and image.
Look at W3Schools Example.

FormMethod

Overrides the method attribute in the form.
Only applicable for the following input types: submit and image.

FormTarget

Overrides the target attribute in the form.
Only applicable for the following input types: submit and image.

FormNovalidate

Makes the form novalidate if it isnt already.
Only works for the submit input type.
